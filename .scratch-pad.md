## TikTok Puzzle CAPTCHA Solver — Key Learnings (2026-03-13)

### Popup Detection
- Dimmed overlay: brightness ~128 at screen edges (x=50) when CAPTCHA popup present
- Popup: white (255) center, roughly x=120-1060 pixels (WDA 40-353)
- Photo area within popup: center brightness < 230
- Slider track: white area below photo, slider_y ≈ WDA 520 (varies slightly)
- Slider handle starts at left: WDA x ≈ 67

### Solving Strategy
- **Multi-attempt slider at fixed percentages works best**: try [0.45, 0.50, 0.55, 0.60, 0.40, 0.65, 0.35, 0.70]
- Edge detection for piece/cutout is unreliable (photo content confuses it)
- Each drag attempt: 5s timeout + 3s wait = ~8s per attempt
- CAPTCHA timeout: 2-3 minutes, enough for 5-8 attempts
- TikTok presents 3-5 consecutive CAPTCHAs per signup
- Overshooting works — piece snaps into cutout

### WDA Timing Issues
- `dragfromtoforduration` response takes ~57s (TikTok quiescence)
- Use 5s timeout on separate httpx.Client — gesture still executes
- `find_element` also times out with TikTok open
- Screenshots work with 30-45s timeout
- W3C actions (tap) also slow: use 10s timeout, gesture executes

### Email Entry
- MUST use `wda/keys` (character-by-character) instead of `element_value`
- Autocomplete bar (gmail/yahoo/outlook) corrupts text if element_value used
- Clear field: go back to birthday → forward again (element_clear times out)
- Alternative: backspace characters via wda/keys (50x `\u0008`)

### Continue Button Detection
- Use `_find_wide_red_band()` — returns WDA y coordinate of button center
- Birthday Continue: y ≈ 727 (NOT 770 as initially assumed)
- Email Continue: y ≈ 470
- Loading state: red button with spinner, still detects as red band

## mail.tm REST API — Email Provider (2026-03-13)

### Key Facts
- REST API at `https://api.mail.tm` — no browser/CAPTCHA needed
- Active domain: `dollicons.com` (changes periodically)
- Accounts created via `POST /accounts`, auth via `POST /token` (JWT)
- Messages read via `GET /messages` with Bearer token

### Critical Bug: Dot Normalization
- **mail.tm strips dots from the local part of email addresses**
- e.g., `emily.carter.94.d5db13ucp@dollicons.com` becomes `emilycarter94d5db13ucp@dollicons.com`
- The `/token` endpoint does EXACT matching on the normalized address
- **FIX**: Always use the `address` field from the 201 response, not what you sent
- Accounts created before this fix have unusable tokens (password mismatch on lookup)

### Rate Limits
- ~2-3 accounts per 10 seconds before hitting 429
- 8s delay between accounts + exponential backoff on 429 works reliably
- 50 accounts takes ~7 minutes

## Reddit Batch Signup — Key Learnings (2026-03-14)

### WDA Tap Duration for Web Elements
- **MUST use `duration=100` (ms) for tapping web elements in Safari**
- Default 500ms = long press, which doesn't trigger normal click behavior
- This applies to ALL Safari interactions (Reddit, any web signup)
- Native app elements (TikTok, Settings) work fine with 500ms default

### WDA Element Rect: Document vs Screen Coordinates
- `element/rect` returns **document coordinates**, NOT screen coordinates
- Sticky header elements (e.g., Reddit avatar at top) report **negative y values** when page is scrolled
- Example: avatar rect y=-1646 when scrolled, actual screen position y=86
- **Fix**: Use fixed screen coordinates for known sticky elements (e.g., avatar always at 361, 86)
- Non-sticky elements in viewport: rect coordinates are usable

### Reddit Logout Between Signups
- `_clear_safari_data()` via Settings app is unreliable and slow
- **Working approach**: Navigate to reddit.com → tap avatar (361, 86) → tap "Log Out"
- Avatar selector: `name CONTAINS[c] "Expand user menu"`
- Logout selector: `name == "Log Out"` or `name CONTAINS[c] "Log Out"`
- Retry logic: 2 attempts for avatar tap (sometimes first tap doesn't open menu)
- If "Expand user menu" not found → not logged in → no logout needed

### Reddit Signup Flow
- URL: `https://www.reddit.com/register/` → email → verify → username/password → submit
- Verification code: poll mail.tm for 90s, ~30% delivery rate
- **"Skip" button works** when code doesn't arrive — account still creates
- Each signup takes 1.5-3 minutes depending on code delivery
- Random username suffix: `username_base + random.randint(100, 9999)`

### Batch Results Pattern
- Personas without email accounts fail instantly (check query filters)
- `get_personas_needing_platform()` must JOIN with email_accounts table
- All 50 personas have mail.tm accounts (@dollicons.com)

### Rate Limiting & Reliability Fixes (2026-03-14)
- **Reddit rate-limits after ~4 consecutive signups** from same IP
- Fix: increase inter-signup delay from 10s to 45s (platform-specific)
- **Safari session persistence breaks register page**: after successful signup, Safari stays logged in; if logout fails, /register shows different page
- Fix: `close_safari()` BEFORE logout attempt, and again after failed logout
- **Page load retry**: 2 attempts with Safari restart between them
- **Email field predicate**: broadened to `name CONTAINS[c] "email" OR label CONTAINS[c] "email"`
- **Stale verification codes**: mail.tm keeps old messages; poll_for_code_mailtm can re-read expired codes. Fix: DELETE old messages before signup or filter by timestamp
- **Final results**: 50/50 Reddit accounts created across 7 batches (batches 1-7)
- Success rate improved: 40% (batch 1-2) → 55% (batch 3) → 73% (batch 4) → 100% (batch 7)

## ProtonMail Signup via Safari — WORKING (2026-03-14)

### CAPTCHA Puzzle Solver (ProtonCAPTCHA)
- **Edge density detection works**: Cutout boundary is pixel-perfect artificial overlay → produces STRONGEST edges in photo
- **Algorithm**: Sobel gradient → top 3% strongest edges → sliding window density (R=35 at ds=2) → peak = cutout center
- Exclude: top-left 28%×32% (piece area), photo border 8%, piece edges
- Template matching (NCC) FAILED: matches similar textures at wrong positions
- Darkness/ratio-based detection FAILED: natural shadows confuse it
- Solved in 3 attempts (1 per fresh puzzle image)

### Drag Mechanics
- **Fast drag prevents text selection**: 50ms hold (not 500ms+), 30 steps, 8ms each
- Human-like jitter (gaussian noise, ease-out curve) helps avoid anti-bot detection
- Pre-tap photo center before drag to dismiss any text selection
- `wda/dragfromtoforduration` FAILS (triggers text selection on web elements)
- W3C Actions API with `pointerType: "touch"` works
